# Задание 1: Заполните контекст проекта

## Цель

Создать полное описание контекста вашего проекта, которое будет использоваться для всех последующих анализов.

---

## Что нужно сделать

1. Откройте файл `context.md` в корне репозитория
2. Заполните все разделы
3. Создайте PR с изменениями

---

## Требования к контексту

### Обязательные разделы

- [ ] О компании (минимум 2-3 предложения)
- [ ] Продукт/услуга (что именно продаёте)
- [ ] Целевая аудитория (кто покупает)
- [ ] Текущая ситуация (где сейчас)

### Желательные разделы

- [ ] Конкуренты (хотя бы 2-3)
- [ ] Цели (на 3 мес и год)
- [ ] Ограничения (бюджет, ресурсы)

---

## Промпт для AI-агента

Используйте промпт из файла `assignments/prompts/01-context-prompt.md`

---

## Критерии проверки

✅ Все обязательные разделы заполнены
✅ Информация конкретная, не абстрактная
✅ Понятно чем занимается компания
✅ Понятно кто клиент

---

## Пример хорошего контекста

```markdown
## О компании
Консалтинговая компания Amoeba, работаем 2 года. Специализируемся на 
growth-экспериментах для B2B SaaS в СНГ.

## Продукт
Онлайн-курс "Логика экспериментов" для предпринимателей МСБ.
14 недель, групповой формат, стоимость 500К рублей.

## Целевая аудитория
- Предприниматели МСБ
- 5-20 лидов в неделю
- Нет выделенного growth-специалиста
- Русскоязычный рынок
```
