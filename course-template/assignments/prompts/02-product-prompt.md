# Промпт: Описание продукта

Используйте этот промпт для создания `texts/product_v1.md`.

---

## Промпт

```
Помоги мне создать детальное описание моего продукта для SWOT-анализа.

Вот контекст моего проекта:
[Вставьте содержимое context.md]

Задавай вопросы по одному, чтобы собрать информацию для каждого раздела.
Копай глубже если ответ абстрактный — нужны конкретные примеры и цифры.

После всех вопросов сформируй документ по структуре:

# [Название продукта]

## Суть продукта
Что это? Какую проблему решает? Для кого?

## Механика
Как клиент использует продукт? Какие шаги?

## Результат для клиента
Что конкретно получает? Какие метрики улучшаются?

## Ценообразование
Сколько стоит? Почему столько? Что входит?

## Отличия от альтернатив
Почему выбрать вас, а не конкурента / самостоятельное решение / ничего не делать?

## Доказательства
Кейсы, отзывы, цифры результатов.

## Честные ограничения
Кому НЕ подойдёт? Что НЕ делаете? Где слабые места?

Начни с вопроса о сути продукта.
```

---

## Ключевые вопросы которые задаст агент

1. **Суть:** Что именно продаёте? Если бы нужно было объяснить за 30 секунд?
2. **Проблема:** Какую боль решаете? Что будет если клиент ничего не сделает?
3. **Процесс:** Как выглядит работа с вами от первого контакта до результата?
4. **Результат:** Какой конкретный результат получает клиент? В цифрах?
5. **Цена:** Сколько стоит? Как пришли к этой цене?
6. **Конкуренты:** С кем сравнивают? Почему выбирают вас?
7. **Кейсы:** Есть успешные примеры? Расскажите историю клиента.
8. **Ограничения:** Кому не подойдёт? Где не сработает?

---

## Советы

- Не приукрашивайте — SWOT всё равно найдёт слабости
- Цифры важнее эмоций: "конверсия выросла на 40%" лучше чем "результаты улучшились"
- Честность про ограничения = сила, а не слабость
